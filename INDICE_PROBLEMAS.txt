SEGURANCA (5)
=============

[1.1] API KEYS EXPOSTAS - lib/supabase.js, lib/tmdb.js
      Severidade: CRITICO
      Arquivo: lib/supabase.js (linha 5-6), lib/tmdb.js (linha 3-5)
      Problema: process.env.NEXT_PUBLIC_TMDB_READ_TOKEN visible no client
      Risco: Requisicoes diretas nao autorizadas a API
      Solucao: Criar API routes como proxy
      Tempo: 1-2 horas
      Prioridade: 1

[1.2] URL INJECTION - lib/tmdb.js
      Severidade: CRITICO
      Arquivo: lib/tmdb.js (linha 16)
      Problema: const url = \\\?\\; sem validacao
      Risco: XSS / Injection attack
      Solucao: Usar URLSearchParams + validacao de endpoint
      Tempo: 30 min
      Prioridade: 2

[1.3] LOCALSTORAGE SEM VALIDACAO - contexts/AuthContext.js
      Severidade: ALTO
      Arquivo: contexts/AuthContext.js (linha 44, 52, 57, 65)
      Problema: localStorage.setItem('activeProfileId', p.id) sem validar
      Risco: XSS via storage injection
      Solucao: Validar UUID pattern antes de salvar
      Tempo: 30 min
      Prioridade: 3

[1.4] VALIDACAO SENHA FRACA - components/Auth/AuthModal.js
      Severidade: ALTO
      Arquivo: components/Auth/AuthModal.js (linha 27-39)
      Problema: Sem validacao de complexidade (maiuscula, numero, simbolo)
      Risco: Senhas fracas aceitas
      Solucao: Regex com requisitos: /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@\$!%*?&])/
      Tempo: 1 hora
      Prioridade: 4

[1.5] ERROR MESSAGES EXPÕEM DETALHES - components/Auth/AuthModal.js
      Severidade: MEDIO
      Arquivo: components/Auth/AuthModal.js (linha 39)
      Problema: setError(err.message) mostra detalhes internos
      Risco: Information disclosure
      Solucao: Mapear para mensagens genéricas
      Tempo: 30 min
      Prioridade: 5
